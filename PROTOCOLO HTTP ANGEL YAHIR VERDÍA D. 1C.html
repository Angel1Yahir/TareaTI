PROTOCOLO HTTP

Explicaré el significado de protocolo http así como su estructura y funcionamiento, metodos de peticion y los codigos de repuesta

¿Que es?...El http (del inglés HyperText Transfer Protocol o Protocolo de Transferencia de Hiper Textos) es el protocolo de transmisión de información de la World Wide Web, es decir, el código que se establece para que el computador solicitante y el que contiene la información solicitada puedan “hablar” un mismo idioma a la hora de transmitir información por la red. Con el http se establecen criterios de sintaxis y semántica informática (forma y significado) para el establecimiento de la comunicación entre los diferentes elementos que constituyen la arquitectura web: servidores, clientes, proxies. Fue creado en 1999 por el World Wide Web Consortium en colaboración con la Internet Engineering Task Force.

Se trata de un protocolo “sin estado”, vale decir, que no lleva registro de visitas anteriores sino que siempre empieza de nuevo. La información relativa a visitas previas se almacena en estos sistemas en las llamadas “cookies”, almacenadas en el sistema cliente.

El http ha pasado por numerosas versiones hasta alcanzar la actual a principios del siglo XXI, llamada HTTP/2. Sus primeros intentos se produjeron en 1991 y arrojaron versiones parciales en 1996, 1999, 2000 y, finalmente, la vigente en 2015



ESTRUCTURA Y FUNCIONAMIENTO

El Protocolo de Transferencia de Hipertexto (HTTP) es el protocolo utilizado para la comunicación entre un cliente (generalmente un navegador web) y un servidor web. Su estructura y funcionamiento se basan en una serie de solicitudes y respuestas. Aquí tienes una descripción general:

1. Cliente y Servidor: En una comunicación HTTP, hay dos partes principales: el cliente y el servidor. El cliente es generalmente un navegador web o una aplicación que solicita recursos, como una página web, y el servidor es una computadora que aloja esos recursos y los entrega al cliente.


2. Solicitud HTTP: El cliente envía una solicitud HTTP al servidor para solicitar un recurso, como una página web o un archivo. La solicitud consta de varios componentes, incluyendo:

-Método: Indica la acción que el cliente desea realizar, como GET para obtener información, POST para enviar datos al servidor, etc.

- URI (Uniform Resource Identifier): La dirección del recurso solicitado.

- Cabeceras HTTP: Información adicional sobre la solicitud, como el tipo de navegador, las cookies, etc.

- Cuerpo de la Solicitud: Opcionalmente, se puede incluir un cuerpo que contiene datos a enviar al servidor, por ejemplo, al realizar una carga de archivo.


3. Procesamiento en el Servidor: El servidor recibe la solicitud y la procesa. Puede acceder a la base de datos, realizar cálculos u otras acciones según la solicitud.


4. Respuesta HTTP: Una vez que el servidor ha procesado la solicitud, envía una respuesta HTTP al cliente. La respuesta también consta de varios componentes, que incluyen:

- Código de Estado: Un número que indica si la solicitud fue exitosa (como 200 OK), redirigida (como 301 Moved Permanently), o si hubo un error (como 404 Not Found).

- Cabeceras de Respuesta: Información adicional sobre la respuesta, como la longitud del contenido, el tipo de contenido, etc.

- Cuerpo de la Respuesta: El recurso solicitado en sí, como una página HTML, una imagen, un archivo de texto, etc.


5. Cierre de la Conexión: Después de enviar la respuesta, la conexión entre el cliente y el servidor se puede cerrar, a menos que se esté utilizando HTTP/1.1 con conexiones persistentes, en cuyo caso se puede reutilizar para solicitudes posteriores.

Este ciclo de solicitud y respuesta es la base del funcionamiento de HTTP. Es un protocolo sin estado, lo que significa que cada solicitud se procesa de forma independiente sin conocimiento de las solicitudes anteriores. Además, HTTP puede ser seguro mediante el uso de HTTPS, que cifra la comunicación entre el cliente y el servidor para proteger la privacidad y la integridad de los datos.


METODOS DE PETICION

HTTP define varios métodos de solicitud que indican la acción que un cliente desea realizar en un recurso en un servidor. Aquí están los métodos de petición HTTP más comunes:

1. GET: Este método se utiliza para solicitar datos de un recurso específico. Cuando se envía una solicitud GET, el servidor devuelve el recurso solicitado, pero no modifica nada en el servidor ni en los datos. Es utilizado principalmente para recuperar información.

2. POST: El método POST se utiliza para enviar datos al servidor para su procesamiento. Por lo general, se utiliza para enviar datos de formularios HTML al servidor, como cuando envías un formulario de inicio de sesión o un formulario de registro. Los datos enviados con POST pueden ser procesados por el servidor y pueden cambiar el estado del servidor o almacenar datos en el servidor.

3. PUT: El método PUT se utiliza para actualizar un recurso en el servidor o crear uno nuevo si no existe. Cuando envías una solicitud PUT, estás enviando datos que deben reemplazar completamente el recurso existente en la ubicación especificada.

4. DELETE: Como su nombre indica, el método DELETE se utiliza para eliminar un recurso en el servidor. Al enviar una solicitud DELETE a una URL específica, el servidor elimina el recurso asociado a esa URL.

5. PATCH: El método PATCH se utiliza para aplicar modificaciones parciales a un recurso en el servidor. En lugar de reemplazar completamente el recurso, como lo hace PUT, PATCH permite realizar actualizaciones parciales.

6. HEAD: El método HEAD es similar a GET, pero el servidor solo debe responder con las cabeceras de la respuesta, sin el cuerpo del recurso. Esto es útil cuando solo se necesita información sobre el recurso, como su tamaño o tipo, sin descargar todo el contenido.

7. OPTIONS: El método OPTIONS se utiliza para obtener información sobre las opciones de comunicación disponibles para el recurso en el servidor. Por ejemplo, puede solicitar qué métodos son compatibles con un recurso específico.

8. CONNECT: El método CONNECT se utiliza para establecer una conexión de red con un recurso, generalmente a través de un proxy. Se utiliza en casos de túneles SSL/TLS a través de un proxy.

9. TRACE: El método TRACE se utiliza para solicitar una traza de diagnóstico del camino de una solicitud. No se usa con frecuencia y puede presentar riesgos de seguridad si se habilita

Cada uno de estos métodos tiene un propósito específico y debe usarse de acuerdo con las convenciones y estándares HTTP. La elección del método adecuado depende de la acción que desees realizar en un recurso en el servidor.


CODIGOS DE RESPUESTA

Los códigos de respuesta HTTP son números de tres dígitos que el servidor web envía en la respuesta a una solicitud HTTP. Estos códigos proporcionan información sobre el resultado de la solicitud y se dividen en cinco clases principales según su significado:

1. Códigos de Respuesta de Información (1xx): Estos códigos son informativos y no indican un resultado final. Son utilizados para comunicaciones previas a la respuesta real. Algunos ejemplos incluyen:

- 100 Continue: El servidor ha recibido la solicitud inicial y está esperando que el cliente continúe con la solicitud.

2. Códigos de Respuesta Exitosa (2xx): Estos códigos indican que la solicitud del cliente fue recibida, entendida y procesada con éxito por el servidor. Algunos ejemplos incluyen:

- 200 OK: La solicitud se ha completado con éxito. La respuesta contiene la información solicitada.

- 201 Created: La solicitud ha resultado en la creación exitosa de un nuevo recurso en el servidor.

3. Códigos de Redirección (3xx): Estos códigos indican que el cliente debe tomar una acción adicional para completar la solicitud. Algunos ejemplos incluyen:

- 301 Moved Permanently: El recurso solicitado se ha movido permanentemente a una nueva ubicación. El cliente debe actualizar su enlace.

- 302 Found (o 303 See Other): El recurso solicitado se ha encontrado, pero se encuentra temporalmente en otra ubicación. El cliente debe redirigirse temporalmente.


Estos son algunos de los códigos de respuesta HTTP más comunes. Cada código tiene un significado específico y ayuda a los clientes a comprender lo que sucedió con su solicitud y cómo deben responder.
